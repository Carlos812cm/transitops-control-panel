# TransitOps Platform v1.8.0

## Public Demo Deployment

TransitOps Platform v1.8.0 completes the first public Full-Stack demo deployment of the project.

This release publishes the Angular control panel, the Express API and a dedicated PostgreSQL demo database so the project can be reviewed from a browser without requiring local setup.

## Main Changes

- Deployed the Express Back-End API to Render.
- Deployed the Angular Front-End to Render using the SSR/hybrid build output.
- Provisioned and validated a Supabase PostgreSQL demo database.
- Applied Prisma migrations to the public demo database.
- Executed a controlled demo seed for users, vehicles, drivers, routes and trips.
- Configured the production Angular environment to consume the public API URL.
- Configured API CORS through `CLIENT_URL` for the public Front-End origin.
- Configured Angular SSR hosting variables for Render host and proxy headers.
- Updated README documentation with public demo URLs, demo credentials and deployment notes.

## Safety Notes

This release is a portfolio demo deployment. It is not a production rollout for real users or real transportation operations.

- No real operational data is included.
- Secrets are stored only in deployment providers.
- `DATABASE_URL`, database passwords and `JWT_SECRET` are not committed.
- Demo credentials are intentionally public and must not be reused outside the demo database.
- The seed process must never be executed against real production data.
