# TransitOps Platform v1.2.0 - Settings Experience & Profile Menu Polish

## Release objective

Improve the Settings experience and user profile menu interactions in TransitOps Platform.

This release focuses on making Settings clearer, more modular and more professional by separating account profile management from language preferences and polishing the authenticated user dropdown interactions.

## Scope

- Redesign `/settings` as a simple Settings overview.
- Replace the current left account summary card with cleaner navigation cards.
- Add a dedicated `/settings/profile` section for account profile management.
- Add a dedicated `/settings/language` section for language preferences.
- Redesign the language preferences panel using the Figma Make reference as visual guidance.
- Polish the navbar user profile dropdown hover, focus and active states.
- Keep logout visually distinct from regular navigation actions.
- Preserve existing authentication, role access, session synchronization and i18n behavior.

## Out of scope

- Back-End changes.
- Prisma schema changes.
- New API endpoints.
- Authentication logic changes.
- New third-party UI or animation libraries.
- Full E2E test setup.
- Deployment configuration.

## Technical approach

- Keep Settings behind the existing authenticated route protection.
- Split the Settings feature into smaller components.
- Reuse `AuthService` for profile, password and avatar actions.
- Reuse `LanguageService` for language selection and persistence.
- Use Angular templates and SCSS transitions instead of external animation libraries.
- Preserve English and Spanish translations.
